（未完成）从零开始使用 gitbook 生成书籍

GitBook 是一个基于 Node.js 的命令行工具，可使用 Github/Git 和 Markdown 来制作精美的电子书。

编写一半的时候突然意识到，这个工具提供了网页端，其不需要进行繁琐的环境搭建工作便可以工作，且具有下面所使用的桌面端的一切优点（事实上官方已经放弃维护桌面端版本了）。如果不熟悉编程，应当优先考虑 网页端。因此这篇文章就写到这里，不再更新了。

为什么是 gitbook？

如果你想要生成一本书，或是将一本书的扫描版（pdf）转换成文字版，但又不了解排版；如果你希望你生成的书籍要能便于分发，便于校对，修改，gitbook 是你的不二之选。我基于如下理由向你推荐 gitbook——

  • gitbook 使用易学易用的** markdown **作为标记语言，你不需要学习排版知识便能够创建丰富的样式。（本文也是使用 markdown 语言编写的）
  • gitbook 能够方便地生成 html，epub 或是 pdf 文件作为输出，你很容易对书籍进行分发，或是将其上传到你的网站或 git pages 等作为个人网站。
  • gitbook 使用 git 进行版本控制，你能够轻而易举地提交修改和进行回溯，你也可以将你的“源代码”上传到 github 以进行分发和协作修改。（当然，你需要掌握 git 的使用，这不是容易啃的料）
  • 书籍的章节使用目录-文件进行组织，便于书籍的结构化和更新。
  • gitbook 提供了丰富的插件，能够让你自定义书籍的样式，提供搜索，浮动目录等功能（对于网页端来说）
  • ……

这里给读者一个 示例，这是 gitbook 官方的博客，其是使用 gitbook 搭建的。

本文主要分为如下部分——

  1. gitbook 环境的搭建

本文假设读者使用 windows 10（X86-64 架构）系统（使用 linux 和 OS X 的读者大约也不需要这样详尽的文章了……）且没有任何编程经验，因此会尽量详尽，力图让没有基础的人也能够上手 gitbook 的使用。在“gitbook 的使用和“书籍的分发”部分中，本文将简单地实践一下将扫描版书籍转换成 gitbook 形式的电子书并生成 pdf 文件或是便于移动端阅读的 epub 文件，以及上传到 github 博客上的全过程。我会试图在描述中多掺杂一些原理性的东西，以期“授人以渔”。毕竟，编程和了解一些计算机的知识在这个时代几乎是必修课了。

gitbook 环境的搭建

gitbook 只需安装** nodejs 环境即可。这里出于之后要将书籍上传到 github 上的需要，也将要求读者安装 git**。

对 powershell 的一些操作

本文章使用 powershell 作为控制台。因为 powershell 默认禁止执行脚本，因此需要进行一些额外操作。可遵循 此文章 进行处理。

git 的环境搭建

git 是一个版本控制工具。它很容易让你对你的代码（在这里是文章）进行版本控制，如更新，修改，添加，回溯等，也能够方便地拉取他人的代码到本地并进行同步或修改操作。在本文里，git 只负责上传你的书籍所生成的网站到 github 中。

你应当前往 git 的 官网，点击32-bit Git for Windows Setup或64-bit Git for Windows Setup（取决于你的 windows 是多少位的，现在的电脑一般来说都是 64 位的）进行下载。安装一路 next 即可。

如果官网下载速度慢，你可以使用 国内镜像，选择最新的版本进行下载。这里给出本文写作时间最新的** 64 位**版本地址 https://npm.taobao.org/mirrors/git-for-windows/v2.30.0.windows.2/Git-2.30.0.2-64-bit.exe。

安装完成后，通过Win+R（Win 是 windows 图标键，这里是要求按住 Win 键，然后按下 r 键）打开“运行”，输入powershell打开控制台（或者右击开始菜单，也能进入 powershell）。输入git --version，查看是否有这样形式的输出——

如果没有，且显示什么命令未找到之类的，查看下下一节。

你也可以按 win 打开开始菜单，然后直接输入cmd，科塔娜会直接帮你找到结果的，如果你启用了她的话。

nodejs 和 gitbook 的环境搭建

nodejs 一般指 node.js。Node.js 发布于 2009 年 5 月，由 Ryan Dahl 开发，是一个基于 Chrome V8 引擎的 JavaScript 运行环境。

在这里，单纯就 gitbook 软件的运行来说，可以认为 nodejs 是 gitbook 的运行平台，就如 windows 是 qq，office 的运行平台一样。因此 nodejs 是必须安装的。

最新版的 nodejs 似乎和当前版本的 gitbook 不兼容，因此这里使用旧版本的 nodejs，你应当下载目录中的 msi 文件并安装。这里给出 64 位的安装包。

安装完成后，你应当输入npm -v，查看是否有这样形式的输出——

如果没有，查看下一节。

你可以输入 node 进入 nodejs 的 REPL 环境（Read，Evaluate，Print，Loop，也就是说，这个环境将不断进行——读取你的输入，进行运算，打印出结果，重复以上步骤——这样的操作），过一把程序员的瘾。nodejs 使用 javascript，一种动态的脚本语言，广泛运用在网页中，但也可以运行在桌面端。gitbook 就是使用这个语言编写的。输入console.log("hello, fucking world!")试试看？这个操作（实际上，是函数调用）读取一个字符串并打印到控制台中。

安装 gitbook 需要使用 nodejs 所提供的包管理工具** npm**。它的用途是从远程直接安装软件到本机中，并提供更新，移除等功能。在安装前，你应当更换到“国内镜像源”，因为官方提供的源慢到让人无法忍受，甚至还可能会超时。

-g 这个参数是不可以省略的，使用该参数安装的软件将直接保存在 PATH 路径下（什么是 PATH 见下一节），这是符合我们的需要的。

现在，输入gitbook fetch。如果安装正确的话，这一步将会安装 gitbook 的最新版本，需要注意的是，这一步非常慢，可能需要数十分钟，不知道该如何解决……

这里的 gitbook 命令和它安装的 gitbook 是什么关系？请联想瓶和酒的关系，一个瓶子可以装多种酒（可以选择使用哪个版本的 gitbook），人要喝酒，则只能拿着瓶子喝。

如果出现了问题

如果出现了问题，九成九的可能是上述软件的路径并未添加到 PATH 中。也就是说，你应当将 git 和 nodejs 的路径添加到 PATH 中，我的 git 和 nodejs 都安装在了默认位置，其中 git 的路径是C:\Program Files\Git\cmd，nodejs 的路径是C:\Program Files\nodejs。关于如何将这些路径添加到 PATH，你可以按下 win 键，输入 env 并回车。

你将打开“系统属性”窗口。点击“环境变量”按钮，找到“系统变量”（“用户变量”也可以）中的 Path 行，双击并将路径添加进新的一行。

PATH 是一个文件路径的列表——当你在控制台下执行命令（其实每个命令都是一个软件，执行命令就是打开这个软件，只不过控制台下执行的软件很多都没有图形界面罢了）时，windows 会遍历 PATH 中所有路径以图找到该命令并执行。

也就是说，如果你的软件没有添加到 PATH 中，Windows 将找不到该软件并报错。当然，你也可以直接给定软件的路径（无论是相对路径——软件的路径相对于你的控制台程序的当前路径，还是绝对路径——顾名思义）以运行没有添加到 PATH 中的程序，这里不考虑这种操作。

gitbook 的使用

到了这一步，你应当搭建好了环境并掌握了 markdown 的基本语法（其实也没什么东西可学），现在终于要开始学习 gitbook 的使用了！

每一个控制台窗口都有一个工作目录，其显示在光标的左边，形如PS C:\Users\aoymykn>，这里的“C:\Users\aoymykn”就是工作目录。下面的 gitbook init 会在工作目录下生成项目文件。你也可以通过cd命令来更改工作目录。

要创建一个 gitbook 项目，你应当创建一个空文件夹并在该文件夹下按住 shift+鼠标右键，选择“在此处打开 Powershell 窗口”（或 cmd 窗口），输入gitbook init并回车，这条命令将初始化一个项目。其会有这样的回显——

这里的 SUMMARY.md 文件是书籍的目录，README.md 文件是书籍的最初页面，可以认为是封面。

你可以随意在 README.md 中输入一些东西。现在，输入gitbook serve。这条命令会在本地启动一个服务器以预览项目。
